1. 음성 기반 주차권 예약 Agent PoC 2026.01 ~ 2026.02

LangGraph, MCP, STT/TTS, Python

"운전 중 주차권 예약"이라는 페인 포인트를 해결하기 위한 Voice AI Agent PoC. 데이터 분석, 아키텍처 설계, 발표까지 주도.

문제 정의

  • 입차시간-결제시간 차이 분석(최근 3개월) → "운전 중 결제 추정 유저" 24.56%
  • 현재 앱 기준: 12단계, 27번 터치, 평균 3분 42초 → 운전 중 매우 위험

설계 및 구현

  • ReAct 패턴 기반 멀티스텝 Agent 아키텍처 설계: User → STT → LLM Agent → TTS
  • MCP 통신으로 주차장 검색/가격 조회/잔여 확인/예약/결제 도구 연결
  • 만차 시 대안 주차장 자동 추천, 맥락 기반 대화 흐름 관리
  • LangWatch 기반 QA 테스트 30개 시나리오 자동 생성 및 검증, 실제 사용성 테스트 수행 및 피드백 수집

성과

  • 소요 시간 3분 42초 → 30초 (87% 감소), 터치 27회 → 1회 (96% 감소), 시선 이탈 12회 → 0회
  • 전사 중간 발표 10분 진행 (라이브 데모 포함)
2. AI 개발 생산성 도구 구축 및 전사 공유 2025.12 ~ 현재

Claude Code, MCP, mem0, LangChain, Qdrant, Python

파트너센터 개발 도중, 기획 문서를 찾는 데 소모되는 시간보다 AI에게 반복 설명하며 낭비되는 토큰 비용이 더 크다는 것을 인식하고, AI 도구를 직접 구축하여 팀 전체에 전파.

문제 정의

  • 프로젝트 기획 문서(PDF/MD/DOCX 수십 건)가 산재 → 필요한 정보를 찾는 데 시간 낭비
  • AI 에이전트가 세션마다 프로젝트 컨텍스트를 잃음 → 매번 같은 설명 반복으로 불필요한 토큰 소모

설계 및 구현

  • 프로젝트 문서 RAG: 하이브리드 검색(벡터 + BM25 → RRF) + Cross-Encoder 리랭킹, Qdrant 벡터DB, PDF 멀티모달 분석
  • mem0 컨텍스트 라우팅: 트리거 키워드 감지 → 3계층 자동 분기(mem0 API → 14개 로컬 메모리 파일 → 원본 문서)로 156개 프로젝트 메모리를 라우팅하여 세션 간 컨텍스트 유지
  • AI 활용 노하우 15개 항목을 사내 위키에 문서화, 팀 내 설치 가이드 작성 및 공유

성과

  • 156개 메모리를 14개 도메인별 파일로 구조화하고 3계층 라우팅으로 AI 에이전트가 세션 간 컨텍스트를 유지
  • 팀원이 자연어로 기획 문서를 즉시 검색 가능 → 문서 탐색 시간 대폭 단축
3. SDD(Spec Driven Development) 기반 AI 코드 생성 시스템 2026.01 ~ 현재

Claude Code, YAML, Markdown

ProductGo(스토리 매핑 도구)로 정리된 기획 스펙을 종합하여, JIRA 티켓 단위로 AI가 개발 코드까지 생성할 수 있도록 "스펙 정의 → 승인 → 코드 생성" 프로세스를 설계. 20개+ JIRA 티켓에 적용.

문제 정의

  • AI에게 코드 생성 요청 시 스펙 없이 바로 코드 출력 → 잘못된 설계가 그대로 반영
  • 단일 프롬프트에 모든 컨텍스트를 담아 ~5,000 토큰 소비 → AI가 일부를 누락

설계 및 구현

  • ProductGo에서 정리된 유저스토리/AC를 기반으로 "스펙 정의 → 승인 → 코드 생성" 3단계 워크플로우 설계
  • AI가 AC/TC/에러코드/API 명세를 먼저 출력하고, 개발자 검토 후 코드 생성
  • 단일 프롬프트를 4가지 작업 유형별 전용 프롬프트로 분리 (API / Entity / Domain / Provider)

성과

  • 하루에 20개+ JIRA 티켓을 SDD 형식으로 전면 재작성 (AC/TC/에러코드 수십 개 단위)
  • 설계 오류를 코드 작성 전에 차단, 스펙이 곧 개발 명세서 → 팀원 독립 개발 가능
  • 토큰 소비 50~60% 절감 (작업당 ~5,000 → ~2,000 토큰)
4. parking-brain: AI 에이전트용 도메인 지식 베이스 설계 2025.12 ~ 현재

Markdown, JSON, Claude Code

방대한 주차 도메인 지식을 AI 에이전트가 적은 토큰으로도 정확히 이해할 수 있도록, 도메인별로 구조화된 지식 베이스를 설계하고 구축.

설계 및 구현

  • 인증/권한, DB 스키마, API 스펙, 도메인 지식, 아키텍처 결정(ADR), 요구사항 등 주제별 디렉토리로 분리하여 필요한 컨텍스트만 선택적으로 로드
  • API JSON(Source of Truth), ERD, 기획 티켓(80개+)을 하나의 레포에서 통합 관리
  • 팀원별 디렉토리(roger/, sebastian/ 등)를 두어 각자의 레포에서 동일한 도메인 지식을 공유

성과

  • 팀원 각자의 AI 에이전트가 동일한 도메인 지식을 참조하여 일관된 코드/문서 생성 가능
  • 방대한 도메인 문서를 구조화하여 AI가 전체를 읽지 않고도 필요한 부분만 참조 → 토큰 효율 극대화

5. 파트너센터 신규 구축 2026.01 ~ 현재

NestJS, TypeORM, MySQL 8.0, JWT, RBAC

주차장 파트너를 위한 통합 관리 플랫폼 신규 구축. 기존 2개 시스템(구어드민, 파트너앱)에 흩어져 있던 주차장 관리, 정산, 주차권 기능을 하나로 통합.

설계

  • RBAC 권한 체계 설계: 사용자에게 역할(Role)을 부여하고, 역할별로 메뉴 단위 기능권한(조회/수정/다운로드/삭제)을 매핑. 데이터권한은 사용자별 담당 주차장/거점 범위를 지정하여 조회 범위를 제한
  • JWT HttpOnly Cookie + SMS OTP + Refresh Token Rotation 인증 설계
  • API 스펙(Source of Truth) 작성 및 OAS3 프리뷰 CDN 배포

구현

  • 기존에 존재하지 않는 테이블을 참조하던 8개 설계 문서를 실제 테이블 기반으로 일괄 재설계
  • 상세설계문서(SDD)를 인수 기준/테스트 케이스/에러코드 단위로 작성 → 팀원이 독립적으로 개발 가능한 수준의 API 명세 확립

성과

  • 현장 파트너 대상 1차 사용성 테스트 수행 → Major 이슈 5건 도출 및 즉시 반영
