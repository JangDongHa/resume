SDD — Spec Driven Development

ProductGo 스토리맵 → SDD 티켓 20+개 재작성 → AI 코드 생성까지 50-60% 토큰 절감

기존 방식 vs SDD

BEFORE — 기존 AI 코드 생성

  • 스펙 없이 "OO API 만들어줘" → 잘못된 설계 그대로 코드화
  • 단일 프롬프트에 모든 컨텍스트 → ~5,000 토큰 소비
  • AI가 정보 누락 → 완성도 저하, 수동 수정 필요
  • 검토 없이 코드 생성 → 설계 오류를 구현 후에야 발견
→

AFTER — SDD 3단계 워크플로우

  • ProductGo에서 정리된 AC/TC가 티켓에 포함 → 스펙이 곧 명세서
  • 작업 유형별 전용 프롬프트 4종 → ~2,000 토큰/작업
  • AI가 먼저 설계 출력 → 개발자 승인 → 코드 생성
  • 설계 오류를 코드 작성 전에 차단

ProductGo 스토리 매핑 INPUT

Goal → Step 분해 — 에픽(파란색)을 Goals로, 기획 유저스토리를 Steps로 매핑. 각 Step 아래에 TC(Test Case) 티켓이 연결되어 테스트 커버리지 확인
전체 백로그 보드 — 9 IN PROGRESS + 34 해야 할 일. 색상별 에픽 구분(파란:내부관리, 빨간:인증, 주황:결제, 초록:상품). 이 스토리맵에서 SDD 티켓으로 변환

1

→ "Y/N/수정사항" 응답
→
3

생성 (Generate)

승인된 스펙 기반 코드 출력

→ Controller / Service / DTO
→ Entity / Repository
→ Exception / ErrorCode
→ Test 코드 포함

AC3: 결제 카드번호는 마스킹 처리된다 (마지막 4자리만 표시)
AC4: 고객 실명은 마스킹 처리된다 (홍*동 형식)

[마스킹 규칙]
 카드번호: "1234567890123456" → "**** **** **** 3456"
 고객명 2글자: 홍*
 고객명 3글자: 홍*동
 고객명 4+글자: 홍**동

[응답 필드]
 제외: phoneNumber, email, internalAdminInfo
 마스킹: cardNumber, customerName
 원본: vehicleNumber, paymentAmount, productName

AI가 생성한 코드 (Kotlin)

          
fun maskCardNumber(cardNumber: String): String {
    val last4 = cardNumber.takeLast(4)
    return "**** **** **** $last4"
}

fun maskCustomerName(name: String): String {
    return when (name.length) {
        0, 1 -> "*"
        2 -> "${name.first()}*"
        3 -> "${name.first()}*${name.last()}"
        else -> {
            val mid = "*".repeat(name.length - 2)
            "${name.first()}${mid}${name.last()}"
        }
    }
}

AI가 바로 코드를 도출할 수 있는 구조적 이유

    

      

        

          
1. 코딩 컨벤션 사전 정의

          

            패키지 구조, 네이밍 규칙, 예외 처리 패턴, 테스트 프레임워크까지 모두 문서화.

            AI는 CODING_CONVENTIONS.md를 참조해 일관된 코드를 생성.
          

        

        

          
2. AC/TC가 곧 구현 명세

          

            인수조건(AC)에 검증 방법까지 명시. 테스트 케이스(TC)에 시나리오/입력/기대결과 포함.

            AI는 AC를 그대로 비즈니스 로직으로, TC를 테스트 코드로 변환.
          

        

        

          
3. API/DB/에러코드 명세 포함

          

            티켓에 HTTP 메서드, 경로, 요청/응답 JSON, DB SQL, 에러코드까지 포함.

            AI가 추측할 필요 없이 명세 그대로 구현.
